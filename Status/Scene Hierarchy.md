# Scene Hierarchy

Current scene:

```txt
Assets/Scenes/SampleScene.unity
```

This document reflects the current implemented MVP state after Prompt04.

## SampleScene Root Hierarchy

```txt
SampleScene
├─ Main Camera
├─ Directional Light
├─ Global Volume
├─ GameManager
├─ BoardRoot
├─ PiecesRoot
├─ PreparationRoot
├─ MovementRoot
├─ InteractionRoot
├─ CooldownRoot
├─ EnemySetupRoot
├─ PrototypeUIRoot
└─ EnemyAIRoot
```

## Root Objects and Components

### Main Camera

Components:

```txt
Transform
Camera
AudioListener
UniversalAdditionalCameraData
```

Purpose:

```txt
Fixed camera for 3D board and 2D sprite pieces.
```

### Directional Light

Components:

```txt
Transform
Light
UniversalAdditionalLightData
```

Purpose:

```txt
Basic scene lighting.
```

### Global Volume

Components:

```txt
Transform
Volume
```

Purpose:

```txt
URP scene volume.
```

### GameManager

Components:

```txt
Transform
GameManager
PlayerHealthService
```

Purpose:

```txt
Owns core game state flow.
Owns player health service.
Transitions through Boot, StageStart, Preparation, Playing, StageClear, and GameOver.
```

Implemented flow:

```txt
Boot
→ StageStart
→ Preparation
→ Playing
→ StageClear / GameOver
```

Key calls:

```txt
StartStage()
StartPreparation()
StartBattle()
NotifyEnemyKingCaptured()
NotifyPlayerKingCaptured()
```

### BoardRoot

Components:

```txt
Transform
BoardManager
BoardView
```

Runtime children:

```txt
Board Tiles
└─ Tile_x_y objects generated by BoardView
```

Purpose:

```txt
Stores board size and occupancy.
Converts between grid and world coordinates.
Generates simple 3D tile visuals.
Applies placement, selection, movement, and capture highlights.
```

Current board default:

```txt
Width = 6
Height = 6
```

Placement rows:

```txt
Player: y = 0, y = 1
Enemy: y = boardHeight - 2, y = boardHeight - 1
```

### PiecesRoot

Components:

```txt
Transform
```

Runtime children:

```txt
Player_* pieces created by PreparationManager
Enemy_* pieces created by EnemySetupManager
```

Purpose:

```txt
Parent object for runtime-created piece GameObjects.
```

Runtime piece components:

```txt
Transform
PieceController
PieceView
PieceCooldown
SpriteRenderer child created by PieceView when needed
```

### PreparationRoot

Components:

```txt
Transform
PreparationManager
PlacementValidator
ManualPlacementController
```

Purpose:

```txt
Handles pre-battle loadout and placement.
Validates loadout cost.
Requires King.
Prevents placement overlap.
Restricts player placement to bottom two rows.
Calls GameManager.StartBattle() only when preparation is valid.
```

Current MVP loadout example:

```txt
King + Rook + Knight + Pawn + Pawn = 7
```

### MovementRoot

Components:

```txt
Transform
PieceMovementService
```

Purpose:

```txt
Calculates valid movement positions for selected pieces.
Delegates movement rules to the current piece movement behavior.
```

Implemented movement behaviors:

```txt
KingMovement
RookMovement
KnightMovement
PawnMovement
```

### InteractionRoot

Components:

```txt
Transform
PieceActionController
BoardInputController
```

Purpose:

```txt
Executes selected piece movement.
Handles capture.
Starts cooldowns after successful movement.
Notifies GameManager when a King is captured.
Routes board tile clicks when legacy Input is enabled.
```

Important current note:

```txt
The project uses Input System-only settings.
BoardInputController legacy Input polling is guarded to prevent runtime exceptions.
Prototype testing currently relies on PrototypeHud buttons and public methods.
```

### CooldownRoot

Components:

```txt
Transform
PlayerGlobalCooldown
```

Purpose:

```txt
Tracks player global cooldown after a player piece moves.
```

Current global cooldown behavior:

```txt
Duration is clamped to 1-3 seconds.
Player pieces cannot move while global cooldown is active.
```

### EnemySetupRoot

Components:

```txt
Transform
EnemySetupManager
```

Purpose:

```txt
Chooses and spawns lightweight enemy setup patterns.
```

Current enemy patterns:

```txt
PatternA_KingRookPawn
PatternB_KingKnightPawnPawn
PatternC_KingRookKnight
```

Spawn validation:

```txt
Enemy pieces must be placed in the top two board rows.
Overlapping enemy placement is rejected.
```

### PrototypeUIRoot

Components:

```txt
Transform
PrototypeHud
```

Purpose:

```txt
Provides editor-testable prototype controls and status display.
```

HUD displays:

```txt
Current state
Player HP
Manual placement selection
Loadout cost
Placed count
Enemy setup name
Enemy count
Global cooldown remaining time
StageClear / GameOver result banner
```

HUD buttons:

```txt
King
Rook
Knight
Pawn
Setup Player MVP
Spawn Random Enemies
Start Battle
Reset Prototype
```

### EnemyAIRoot

Components:

```txt
Transform
EnemyAIController
```

Purpose:

```txt
Runs lightweight reactive enemy AI during Playing state.
Uses existing movement, board, capture, cooldown, and game state systems.
```

Current AI behavior:

```txt
Capture Player King immediately.
Move each Enemy Pawn once during opening if possible.
Prefer captures by non-King enemy pieces.
Prefer random movement by non-King enemy pieces.
Use Enemy King movement only as fallback unless it can capture Player King.
Wait if no valid movement exists.
```

## Runtime Object Creation

### Board Tiles

Created by:

```txt
BoardView.GenerateBoard()
```

Each generated tile has:

```txt
Transform
MeshRenderer
BoxCollider
BoardTileView
```

Tile names:

```txt
Tile_0_0
Tile_1_0
...
Tile_5_5
```

### Player Pieces

Created by:

```txt
PreparationManager.TryPlaceSelectedPiece(...)
PrototypeHud.SetupPlayerMvpLoadoutAndPlacement()
ManualPlacementController.HandleTileClicked(...)
```

Name pattern:

```txt
Player_{DisplayName}_{LoadoutIndex}
```

### Enemy Pieces

Created by:

```txt
EnemySetupManager.SpawnRandomSetup()
EnemySetupManager.SpawnSetup(...)
PrototypeHud.SpawnRandomEnemies()
```

Name pattern:

```txt
Enemy_{DisplayName}
```

### Enemy AI Actions

Executed by:

```txt
EnemyAIController
EnemyMoveSelector
PieceActionController.TryMoveSelectedPiece(...)
```

Enemy AI action logs:

```txt
Enemy AI opening pawn move
Enemy AI capture
Enemy AI random move
Enemy AI waits
```

## Current Scene Test Flow

Recommended editor flow:

```txt
1. Enter Play Mode
2. Confirm state is Preparation
3. Click Setup Player MVP
4. Click Spawn Random Enemies
5. Click Start Battle
6. Observe enemy AI movement during Playing
7. Select or invoke player piece movement through prototype methods
8. Capture Enemy King to reach StageClear
9. Capture Player King enough times to reach GameOver
10. Reset Prototype to replay
```

Current verified results:

```txt
Board appears in Play Mode
Prototype HUD appears in Play Mode
Player MVP setup places pieces on valid rows
Random enemy setup places enemies on top rows
2D sprite pieces appear above the 3D board
Enemy AI moves automatically during Playing
Enemy AI uses existing movement and capture rules
Enemy AI respects individual piece cooldowns
Enemy AI can capture player pieces
Enemy AI prioritizes Player King capture
Enemy AI moves Enemy Pawns once during opening when possible
Enemy AI reduces early Enemy King aggression
Enemy King capture reaches StageClear
Player King capture damages player HP
Cooldowns start after successful movement
```
